Version Control System
Definition

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. Version control also known as revision control is associated with a timestamp and the person making the change. Revisions can be compared, restored and sometimes merged together. A Version Control System is a very wise thing to use if you are a graphic or web designer as it allows you to revert files back to a previous state and makes the files easily recoverable if you were to lose the files and also makes it easier for a group of developers to work together easily.

Git
Definition

Git is a free and open source distributed version control system created by Linus Torvalds in 2005. It is designed to manage everything from small to very large projects with speed and efficiency. Git is a source code management system. It takes a "peer-to-peer" approach to version control as opposed to a "Client-Server" approach like version control systems like Subversion Git allows a team of people to work together, all using the same files helping the team manage with the confusion that tends to happen when multiple people are editing the same files. Git allows you access to all versions of all files in Git repository at any time, it’s almost impossible to lose any part of a code.

Git Repository
Definition

Git stores this information in a data structure called a repository. A Git repository contains, among other things, the following: A set of commits objects and a set of references to commit objects, called heads. The Git repository is stored in the same directory as the project itself, in a subdirectory called git. . In order to create a repository, create a directory for the project if it doesn’t exist, enter it, and run the command “git init’. Then you must create a commit, in order to do so you first need to tell Git which files to include, with “git add .” . then call “git commit” to create the commit. Once you are done creating a commit you will want to push to file to a remote repository , to do so type the command “git push “.

GitHub
Definition

GitHub is a web-based Git repository hosting service that offers all of the distributed revision control and source code management. The trademark mascot of GitHub is Octocat, an anthropomorphized cat with octopus limbs, done in a manga style. Unlike Git, which is strictly a command-line tool, GitHub provides a web-based graphical interface and desktop as well as mobile integration. It allows you access control and several features such as feature requests, task management, wikis and bug tracking for every project. In addition to source code, GitHub supports the following formats and features:

    Pull requests with code review and comments
    Documentation, including automatically-rendered README files in a variety of markdown-like file formats.
    Commits history
    Issue tracking with labels, assignees and a search engine
